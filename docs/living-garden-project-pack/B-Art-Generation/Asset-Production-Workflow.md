# Living Garden Asset Production Workflow

## Stage 1 — Visual Calibration

Generate only:

- sprout
- white daisy
- lavender
- rounded stone
- cream rabbit

Use the same:

- Master Asset Prompt
- reference image
- lighting direction
- camera angle
- saturation level
- image-generation model
- generation settings where available

## Stage 2 — Founder Review

Place all five images next to each other.

Review:

- Do they look like the same illustrator created them?
- Are the colors rich enough?
- Are any assets too pale?
- Are any assets too cartoonish?
- Do all assets use similar lighting?
- Are their silhouettes clear at mobile size?
- Do the plant, animal and stone belong to the same world?

If the answer is no, revise the Master Prompt or regenerate before continuing.

## Stage 3 — Core Asset Production

After calibration approval, generate the remaining core assets.

Produce assets in small batches by category:

- plants
- natural elements
- animals and insects
- garden objects

Do not generate all categories in one uncontrolled batch.

## Stage 4 — File Preparation

For each approved asset:

- preserve transparent background
- remove white edge contamination
- crop excess empty space without cutting the object
- preserve safe transparent padding
- use the exact required filename
- export as PNG
- keep a high-resolution master copy
- prepare an optimized web copy later

## Stage 5 — Technical Integration

Only approved files should be copied into the application asset folders.

Claude Code should:

- map the approved filename to the existing asset registry
- preserve the existing rendering engine
- preserve Day 1–35 progression
- use Founder Preview for validation
- avoid redesigning the garden
