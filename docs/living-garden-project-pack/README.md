# Living Garden Project Pack

This folder separates Living Garden work into three areas.

### Folder A — Claude Development

**Purpose:**

Technical implementation, rendering structure, asset registration, day unlock logic and framework development.

**Status:**

Sprint 4 is already completed.

This folder is for development records and future technical sprints.

### Folder B — Art Generation

**Purpose:**

Instructions for creating formal Living Garden artwork and transparent PNG assets.

This folder is **not** a Claude Code development prompt.

Its prompts are intended for image-generation tools such as ChatGPT Image or another visual-generation tool.

### Folder C — Product Planning

**Purpose:**

Defines what appears in the garden, when each element appears and how the garden becomes progressively richer from Day 1 to Day 35.

This folder is intended for founder and product review.

---

## Workflow

1. Product planning decides what appears.
2. Art generation creates the required visual assets.
3. Claude Code connects approved assets to the existing engine.
4. Founder Preview is used to review Day 1–35.
5. Only approved assets are used in production.

---

> **Rule:** Never mix coding instructions, image-generation prompts and product-planning decisions inside one prompt.
